## Round 2: 「我年薪四十萬」判不出來（TASK.017）

- 重現：僅說年薪 → 追問縣市/狀況/收入三題後回「資訊不足 + 1957」。根因：(1) 金額無 pattern、income 追問的自由輸入金額也被丟棄；(2) 提問用全域頻率排序，租補（全國性）缺的租約/年齡永遠排不到，MAX_QUESTIONS=3 先耗盡。
- 修法：`parse_income_amount`（中文數字轉換含 3萬5/三萬五 縮寫；年薪÷12；範圍 1k–10M）；租補所得條件 `any(income_status ∈ 4 tokens, monthly_income lte 46545)`；`_pick_next_field` 改「hit_conditions 非空的服務優先 → 最少缺欄 → 跨服務共用度」；MAX_QUESTIONS 5；新增 monthly_income 追問題（問句固定月單位，裸金額可解析）+ FIELD_LABELS。
- 門檻查證：115 年最低生活費——臺北市 20,744、臺灣省 15,515（全國最低）×3 = 46,545 保守門檻；高於門檻不判 unlikely，退回追問身分別。來源記於 ADR-0008 References。
- 中途發現 `test_chat_api` 寫死 `turns <= 3`，改 import `MAX_QUESTIONS`。第一版 closest-only 策略讓 Q1 跑去問長照（只差一欄），加上 hit-first 後才收斂：年薪 → Q1 年齡、Q2 租約 → 租補 possible。
