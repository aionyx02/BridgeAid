# Session Log: 2026-07-14 - Shawn

## Summary

- Modularized the backend FastAPI entrypoint and the static web demo so future
  feature work can attach to focused modules instead of growing large entry files.
- Hardened the conversation state machine and Ollama fallback path so unclear
  answers, repeated follow-ups, and LLM timeouts do not make the chat feel stuck.

## Actions Taken

- Created `backend/app/runtime.py` for process-local dependencies, cached rules,
  the conversation manager, reminder service, and LINE reply adapter.
- Split FastAPI endpoints into `backend/app/api/` routers for health,
  recommendations, chat, reminders, services/source trace, and LINE webhook.
- Added `backend/app/factory.py` so `backend/app/main.py` is now a small
  application entrypoint.
- Moved LINE reminder postback handling into `backend/app/line/reminder_flow.py`
  with explicit service/reminder dependencies.
- Added `backend/app/service_catalog.py` for pure service response mappers.
- Split `demo/app.js` into browser-native ES modules: `core.js`, `render.js`,
  `chat.js`, `services.js`, and `reminders.js`.
- Updated tests to patch the shared runtime LINE client and to verify a demo JS
  module is served.
- Added explicit conversation states for collecting data, awaiting a field
  answer, and result-ready follow-up handling.
- Added one retry for unrecognized pending-field answers before skipping that
  field, preserving state instead of silently burning the question.
- Added result follow-up replies for document, application-process, and source
  questions after recommendations are shown.
- Added an Ollama failure cooldown so a timeout or invalid response temporarily
  uses the deterministic parser instead of repeatedly waiting on the LLM.
- Added regression tests for fuzzy pending answers, unclear-answer retry,
  post-result document/process questions, and Ollama cooldown.

## Detailed Notes

- Local git branch created: `feature/modularize-codebase`. The project has no
  local or remote `dev` branch, so the branch was created from the clean `main`
  branch while preserving the project `feature/<name>` convention.
- Backend behavior stayed route-compatible: `/healthz`, `/recommend`, `/chat`,
  `/reminders`, `/services`, `/services/{id}/source`,
  `/services/{id}/process`, and `/line/webhook` are still registered through
  the same `app.main:app` entrypoint.
- The largest backend entrypoint was reduced to app construction only. Shared
  runtime state now has one import location for tests and new adapters.
- The demo keeps the existing `type="module"` script tag; the browser now loads
  `app.js` as an orchestrator that imports smaller modules by responsibility.
- Conversation follow-up questions now distinguish between accepted, skipped,
  and unrecognized answers. Unrecognized replies are retried once, which avoids
  both infinite re-asking and prematurely losing the pending field.
- Repeated recommendation dedupe now uses a result signature and only suppresses
  unchanged results when the profile did not change. The result-ready state can
  still answer "需要文件", "申請流程", and "來源" from stored recommendation context.
- Ollama intent extraction remains untrusted and optional; after a handled
  failure it cools down for 60 seconds and falls back to deterministic parsing.
- Follow-up audit found remaining bugs to prioritize: LINE webhook invalid JSON
  returns 500 instead of a controlled 400, pending-field answers do not accept
  common Chinese forms such as `四十歲` and `無`, the demo cancel-reminder path
  treats failed HTTP responses as success, and LINE reply API failures are not
  surfaced by the HTTP client adapter.
- Connectivity check: remote Docker over SSH works with `docker -H
  ssh://aionyx-vps`; remote `bridgeaid` is running on `127.0.0.1:8091->8080`
  and returns `/healthz` 200. Local ngrok points to `localhost:8000`; after
  restarting the local Uvicorn process, both local and ngrok `/healthz` return
  200.
- Fixed the audited bugs: LINE webhook now returns 400 for malformed/non-object
  JSON, pending answers accept `無` and Chinese numeral ages such as `四十二歲`,
  demo reminder cancellation now checks HTTP failures, and LINE reply messages
  raise for non-2xx API responses.
